# False Binary from a Shared Hidden Premise

## What This Pattern Is

Two sides argue X-vs-Y indefinitely; the real culprit is a premise *both*
silently share. Surfacing and dropping it **dissolves** the dispute —
there was no fact to settle — rather than resolving it. The binary is an
*artifact* of the shared assumption: remove the premise and the question
doesn't get an answer, it loses its presupposition. A structural move
(re-framing the question-space), not a rhetorical one.

in the excogitations archive), status "reconcile with local library" —

## Recognizing When This Applies

**Conditions:**
- A debate that is old, intelligent on both sides, and unresolvable — the
  longevity itself is the tell that no settling fact exists.
- You're being pressed to pick a side and both options feel subtly wrong.
- Design deadlocks: two architectures argued for weeks may share a wrong
  requirement.

**Not this pattern:**
- Genuine empirical disputes — most binaries are real and one side is
  wrong; reach for evidence first, dissolution only when evidence keeps
  failing to settle it.
- Using "you both share a premise" as a debate-exit trick without naming
  the premise and showing the question fails without it.

## Cross-Domain Verification

**Mechanism check:** presupposition failure — a question can be
well-formed only relative to its presuppositions; remove one and the
question is neither true nor false but *void*. Shared logical structure,
not vocabulary.

**Domains verified:**
- Extended-mind debate — both sides assume determinate cognitive
  *ownership*.
- The cogito's critics — being = thought assumed by Descartes *and* many
  attackers.
- Nature-vs-nurture — both assume the two are separable.
- Free-will debates — both sides assume the libertarian/deterministic
  dichotomy exhausts the options.

synthesis):
- `parallel-not-opposed` — rivalry manufactured by forcing *orthogonal*

- `label-bundling-dead-axis` — why a dead axis *persists* (identity

- `frame-pushback` — the interaction choreography (refuse the frame across

## What Makes It Work

1. **Name the premise explicitly** — "you both assume P" only counts when P
   is stated sharply enough that each side recognizes it as theirs.
2. **Show the void, not a winner** — the payoff is the question failing,
   not side X being right after all; if a side wins, this wasn't the
   pattern.
3. **Offer the post-dissolution question** — dropping P usually exposes a
   better-formed successor question; hand it over, or the vacuum refills
   with the old binary.

## Testing This Pattern

Before relying on this:
1. **Baseline:** Engage a stuck binary WITHOUT the move (pick a side, argue).
2. **With pattern:** On another, hunt the shared premise first.
3. **Compare:** Did premise-hunting shorten the path or produce the better question?
4. **Pressure:** Does it work when rushed?

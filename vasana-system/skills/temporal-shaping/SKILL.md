# Temporal Shaping

## Core Tenet

This is a Vasana - an interaction choreography, not an AI skill. It describes a dance that emerges between minds, not what either party does alone.

---

## What This Pattern Is

**Everything that unfolds over time has a shape.** Sounds have envelopes. Conversations have arcs. Projects have phases. Learning has curves.

### Two Temporal Models

**1. ADSR (Linear, One-Shot)** - The synthesist's envelope:
- **Attack:** How does it begin? Sharp or gradual?
- **Decay:** How does initial energy settle?
- **Sustain:** What persists during the main engagement?
- **Release:** How does it end? Abrupt or fading?

**2. Reich Cycle (Cyclic, Must Complete)** - From Wilhelm Reich via Christopher S. Hyatt:

```
Tension → Charge → Discharge → Relaxation
    ↑________________________________↓
              (cycle repeats)
```

- **Tension:** Build-up, anticipation, problem recognition
- **Charge:** Peak energy, full engagement, maximum effort
- **Discharge:** Release, breakthrough, the drop, action
- **Relaxation:** Return to baseline, integration, satisfaction

**The critical difference:** ADSR can stop at any stage. The Reich cycle MUST complete for satisfaction. **If you can't complete the cycle, it's a failure.**

This applies universally:
- A drum hit that doesn't complete its pitch-drop cycle sounds wrong
- A conversation stuck in "charge" without discharge leaves both parties frustrated
- A project in eternal "sustain" without release feels endless and joyless
- Learning without the "discharge" of application stays theoretical forever

**Origin:** This vasana emerged from studying MindMeld ShapeMaster (a modular synthesizer module) and Christopher S. Hyatt's work on Reich's tension cycles, recognizing that temporal shaping applies universally to any process that must complete to succeed.

## Recognizing When This Applies

**Conditions:**
- Designing something that unfolds over time
- Something feels "off" in its pacing but you can't articulate why
- Need to structure phases of a process
- Cross-domain problem where audio/visual/cognitive analogies might help

**Not this pattern:**
- Static, timeless decisions
- Binary choices (yes/no)
- Purely spatial problems

## The Pattern

### Opening: Name the Dimensions

Before shaping, identify what's being shaped:

**In audio:** Amplitude (volume), Pitch (frequency), Timbre (harmonic content)
**In conversation:** Intensity (engagement), Direction (topic), Depth (abstraction level)
**In project:** Effort (resources), Scope (features), Risk (uncertainty)

**The move:** Ask "What are the independent dimensions I can shape over time?"

### Development: Sketch the Envelope

For each dimension, sketch the intended shape:

```
     ╭─╮
    ╱   ╲____
   ╱         ╲
  ╱           ╲
─╯             ╰──

Attack | Decay | Sustain | Release
```

**Questions to answer:**
- How sharp is the attack? (Instant engagement or gradual warmup?)
- Where does decay settle? (How much initial energy dissipates?)
- What sustains? (The core that persists)
- How long is release? (Abrupt end or graceful fade?)

### Landing: Test the Shape

Apply the shape consciously. Then ask:
- Did the actual trajectory match the intended shape?
- Where did it deviate?
- Was the deviation a problem or a discovery?

## Cross-Domain Applications

### Conversation Design

| Phase | Sound Analogy | Conversation Reality |
|-------|---------------|---------------------|
| Attack | Sharp transient | Hook, provocative question |
| Decay | Initial falloff | Settling into topic |
| Sustain | Core body | Main exploration |
| Release | Tail/reverb | Resolution, next steps |

**Mismatch example:** "This meeting felt rushed"
**Diagnosis:** Attack was too long (too much preamble), leaving no time for sustain.

### Learning Progression

| Phase | Sound Analogy | Learning Reality |
|-------|---------------|-----------------|
| Attack | Punch | Initial concept introduction |
| Decay | Settling | First practice, initial struggle |
| Sustain | Body | Deliberate practice, reinforcement |
| Release | Fade | Transfer to new context, letting go of scaffolds |

**Mismatch example:** "Students forget everything"
**Diagnosis:** No sustain phase - moved from attack directly to release.

### Project Phases

| Phase | Sound Analogy | Project Reality |
|-------|---------------|-----------------|
| Attack | Transient | Kickoff energy, initial momentum |
| Decay | Initial drop | Reality setting in, scope clarification |
| Sustain | Core signal | Steady execution |
| Release | Tail | Handoff, documentation, transition |

**Mismatch example:** "Projects always drag at the end"
**Diagnosis:** No designed release - sustain just fades indefinitely.

## Advanced Shapes

### Multi-Stage (MSEG)

Beyond ADSR, use arbitrary stages:

```
     ╭╮  ╭─╮
    ╱  ╲╱   ╲___
   ╱            ╲
──╯              ╰─

[1][2][3] [4] [5]
```

**Application:** Project with multiple sprints, conversation with multiple topics, learning with multiple concepts.

### Loop Points

Some processes need to cycle:

```
      ╭──────╮
     ╱   ↺   ╲
────╯         ╰────
     │ loop │
```

**Application:** Iterative refinement (red-green-refactor), practice cycles, review processes.

### Sidechain (External Shaping)

One process shapes another:

```
Shaper:  ▄ ▄ ▄ ▄ (external input)
Shaped:  ╲╱╲╱╲╱╲ (internal process responds)
```

**Application:** Productive friction (external challenges shape internal growth), deadlines (external constraints shape work rhythm), feedback (external input shapes direction).

## What Makes It Work

1. **Naming dimensions explicitly** - Can't shape what you haven't identified
2. **Sketching before executing** - Visual/conceptual shape precedes implementation
3. **Recognizing mismatches** - "Feels off" usually means shape mismatch
4. **Cross-domain scanning** - Sound design vocabulary illuminates other domains

---

## Testing This Vasana

Before relying on this:
1. **Baseline:** Design a process WITHOUT explicit temporal shaping thinking
2. **With pattern:** Design another, consciously applying envelope concepts
3. **Compare:** Did explicit shaping improve the design?
4. **Pressure:** Does it work for quick decisions or only deliberate planning?

**Honest note:** This pattern works best for medium-to-long processes (minutes to months). For micro-interactions (seconds), it may be overkill. For very long processes (years), other factors dominate.

# Cui Bono Investigation Methodology v4.0

## Power Structure & Ethical Analysis Framework

*Architecture note: "Lenses" are analytical frames constructed sequentially, not parallel processes. Current Claude constructs and applies them one at a time within a single context. If future systems support genuine parallel architectures, lenses should become agents with concurrent investigation and synthesis capabilities.*

*Epistemological note: This methodology synthesizes techniques from Western intelligence tradecraft, Russian independent journalism, Chinese analytical frameworks, Global South investigative networks, and critical/indigenous research paradigms. No single tradition is treated as baseline—each reveals blind spots in others. Anti-imperialist conclusions should EMERGE from rigorous multi-perspective analysis, not from ideological prescription.*

---

## 1. Framework Clarification - ALWAYS FIRST

**Before launching ethical analysis, establish user's framework through clarifying questions.**

### 1.1 Core Framework Questions

| Dimension | Questions to Ask |
|-----------|------------------|
| **Weighting** | Which concerns weigh heaviest? (weapons > labor > environment, or different hierarchy?) |
| **Thresholds** | Are some violations absolute dealbreakers, or seeking "least problematic" graduated assessment? |
| **Supply Chain Depth** | Trace to end-users? Both upstream (sourcing) and downstream (customers)? |
| **State Ownership Stance** | Treat state ownership as neutral, positive, or negative? Examine state actor behavior separately? |

### 1.2 Imperial Definition - CRITICAL

**Which actors/structures count as "imperial"? This question determines entire analytical direction.**

| Framing | Implication |
|---------|-------------|
| **US/Western only** | Analysis focuses on NATO, Five Eyes, Bretton Woods institutions, dollar hegemony |
| **All major powers** | Includes China (BRI, debt diplomacy, Xinjiang), Russia (military interventions, energy coercion), regional powers (Saudi, Iran, Turkey) |
| **Structural** | Any entity enabling extraction, surveillance, or coercion regardless of national origin |
| **Historical-materialist** | Centers capital accumulation patterns; state actors analyzed through class relations |

**Do not assume user's position. Ask explicitly.**

Related questions:
- Are weapons equally problematic regardless of who they serve, or weighted by recipient?
- Is state ownership inherently problematic, neutral, or potentially positive (democratic control of resources)?
- How do you weigh historical colonial extraction vs. current behavior?

### 1.3 Analyst Positioning (Strong Objectivity Requirement)

Before analysis, explicitly acknowledge:
- What is the analyst's own geopolitical position?
- Whose interests might this analysis serve?
- What frameworks is the analyst trained in, and what might they systematically miss?

This is not disclaimer theater—it structures what counter-perspectives to actively seek.

## 2. Investigation Structure

### Phase 1: Decomposition
Break the unified question into domain-specific investigations. Each lens sees only its domain—this separation prevents premature synthesis and ensures coverage.

### Phase 2: Lens Construction & Application
Construct each relevant lens from domain templates. Apply sequentially using the Claims → Contradictions → Resolution structure.

### Phase 3: Recomposition
Not aggregation. Ask: *"What pattern emerges from the combination that no single lens could reveal?"*

Synthesis produces qualitatively new understanding—emergent patterns, system interactions, trade-offs invisible from any single perspective.

## 3. Multi-Lens Analysis

### 3.1 Lens Application Protocol (per domain)

For each lens, follow the **Claims → Contradictions → Resolution** structure:

```
1. CLAIMS
   Document entity's self-presentation and conventional narrative.
   What do they say about themselves? What does consensus assume?
   
   Sources: Corporate communications, investor presentations, friendly media coverage,
   industry analyst reports, official regulatory filings (stated purposes)

2. CONTRADICTIONS
   Actively seek counter-evidence through multiple methods:
   
   A. DIRECT SEARCH
      - Search for evidence explicitly contradicting claims
      - Use adversarial sources (short-seller reports, hostile media, labor unions,
        affected communities, competitor analyses)
      - Check regulatory enforcement actions, lawsuits, settlements
   
   B. DEDUCTIVE VERIFICATION
      - "If claims true, X must also exist" — verify X exists
      - "If business model is Y, financial patterns should show Z" — check Z
      - Map what SHOULD be observable if narrative is accurate
      - Absence of expected evidence IS evidence
   
   C. HYPOTHESIS FALSIFICATION (from ARIJ method)
      - Treat claims as hypothesis, not fact
      - "What would disprove this?" — actively search for it
      - Design searches to find disconfirming evidence, not just confirming
   
   D. MULTI-TRADITION CROSS-CHECK
      - How would Chinese contradiction analysis frame this entity?
      - What would dependency theory reveal about center-periphery dynamics?
      - What do affected communities (indigenous, labor, local) know that
        management doesn't? (Feminist standpoint method)
      - Whose knowledge has been made invisible? (Santos' sociology of absences)

3. RESOLUTION
   Assessment integrating tension between claims and contradictions.
   Not "who wins" but "what's actually true given both."
   
   State explicitly:
   - What evidence supports claims (with quality rating)
   - What evidence contradicts claims (with quality rating)
   - What remains genuinely uncertain
   - What additional investigation would resolve uncertainty
```

## 4. Evidence Evaluation Standards

### 4.1 Multi-Tradition Evidence Framework

Different analytical traditions define "evidence" differently. Make explicit which standard applies:

| Tradition | Evidence Priority | Credibility Markers | Limitations |
|-----------|-------------------|---------------------|-------------|
| **Western OSINT** | Documentary, verifiable, attributable | Transparent provenance, multi-source triangulation | Privileges literate/documented over oral/experiential |
| **Russian Independent Journalism** | Verifiable data (including black market acquisition) | Methodology transparency, partner verification, legal survival | Access constraints, personal risk |
| **Chinese State Frameworks** | Comprehensive multi-domain assessment | State media credible for *official position* (not necessarily fact) | Ideological overlay, access limitations |
| **Global South Journalism** | Legally bulletproof, hypothesis-verified | Cross-border corroboration, prosecutorial review | Resource constraints, safety risks |
| **Indigenous Methods** | Oral tradition, community testimony, relational knowing | Community validation, elder verification | May lack documentary trail |
| **Feminist Standpoint** | Marginalized experience, embodied knowledge | Testimony from affected parties | May conflict with institutional sources |
| **Liberation Theology** | Lived experience of the oppressed | Collective discernment | Explicitly normative |

### 4.2 Evidence Quality Categories

**Categorize all claims explicitly using modified Admiralty System:**

**Source Reliability (A-F):**
- A: Completely reliable (verified track record, no known deception)
- B: Usually reliable (good track record, occasional errors)
- C: Fairly reliable (limited track record, some concerns)
- D: Not usually reliable (poor track record, known issues)
- E: Unreliable (demonstrated deception or incompetence)
- F: Reliability cannot be judged (new or unknown source)

**Information Credibility (1-6):**
- 1: Confirmed by independent sources
- 2: Probably true (consistent with known information, logical)
- 3: Possibly true (not confirmed, not contradicted)
- 4: Doubtful (inconsistent with known information)
- 5: Improbable (contradicted by known information)
- 6: Truth cannot be judged (insufficient basis)

**Critical insight: Source reliability and information credibility are INDEPENDENT.** An A-rated source can deliver 4-rated information. Disinformation often comes from usually reliable sources.

### 4.3 Practical Evidence Categories

For operational use, translate to:

| Category | Standard | Examples | Citation Format |
|----------|----------|----------|-----------------|
| **Verified** | Primary documentation, multiple independent confirmations | SEC filings, court records, regulatory findings, audited financials | Direct citation with document ID |
| **Credible Allegation** | Multiple independent sources, reputable organizations, methodology transparent | HRW/Amnesty reports, academic peer-reviewed studies, quality investigative journalism | Source + methodology note |
| **Contested** | Single source, disputed, potentially motivated | Short-seller reports, government accusations without evidence, partisan media | Source + motivation flag + counter-sources |
| **Speculation** | Logical inference without documentation | "They probably also..." pattern-based inference | Explicit speculation flag |

**Never present speculation as fact. Always distinguish verified from alleged. Note source motivations.**

### 4.4 Deductive Evidence and Absence

When expected evidence is absent, note explicitly:

> "If X were true, Y should exist. Y was not found despite searching [specific sources]. This absence is evidence against X, weighted [high/medium/low] based on [how definitively Y should have appeared]."

Example:
> "If Centrus operates purely civilian enrichment, DOE/NNSA weapons-complex contracts should be absent from federal procurement databases. Search of USAspending.gov for Centrus/USEC contracts 2015-2025 found [X contracts totaling $Y with weapons-related purpose codes]. This contradicts purely-civilian claims with high confidence."

## 5. Specific Analytical Techniques

### 5.1 Analysis of Competing Hypotheses (ACH)

From CIA tradecraft, adapted for corporate investigation:

1. **Brainstorm all possible hypotheses** about the entity's true nature/behavior
   - Include hypotheses you consider unlikely
   - Seek input from analysts with different frameworks
   
2. **List significant evidence** for and against each hypothesis
   
3. **Construct matrix**: Evidence (rows) × Hypotheses (columns)
   - Rate each cell: Consistent (C), Inconsistent (I), Neutral (N)
   
4. **Work ACROSS the matrix** (one evidence item against all hypotheses)
   - NOT down (one hypothesis against all evidence)
   - This prevents confirmation bias
   
5. **Focus on disconfirmation**
   - Evidence that rules OUT hypotheses is more valuable than evidence that supports
   
6. **Identify diagnostic evidence**
   - Evidence consistent with only one hypothesis has HIGH value
   - Evidence consistent with all hypotheses has NO diagnostic value
   
7. **Sensitivity analysis**
   - Which evidence items are critical? What if they're wrong?
   
8. **Monitor rejected hypotheses**
   - Develop indicators that would signal weaker hypotheses becoming stronger

### 5.2 Contradiction Analysis (矛盾分析)

From Mao's dialectical method, operationalized:

1. **Identify all contradictions** within the entity/situation
   - Capital vs. labor
   - Growth vs. sustainability  
   - Stated mission vs. revenue sources
   - Short-term vs. long-term interests
   - Different stakeholder interests

2. **Distinguish principal from secondary contradictions**
   - Which contradiction's resolution is decisive for the entity's trajectory?
   - Resolution of principal contradiction transforms the whole situation

3. **Identify principal aspect** of principal contradiction
   - Which side of the contradiction is dominant?
   - Which side's development will determine resolution?

4. **Track transformation**
   - Principal and secondary contradictions can switch positions
   - What conditions would cause transformation?

**Application example:**
> XP Inc principal contradiction: platform growth vs. regulatory sustainability. Currently growth-dominant (management prioritizing expansion over compliance). Secondary contradiction: retail investor access vs. institutional quality standards. If regulatory action intensifies, contradictions may transform—compliance becomes principal, forcing business model changes.

### 5.3 Hypothesis-Based Inquiry (ARIJ Method)

From Arab Reporters for Investigative Journalism:

1. **Story is only a hypothesis until verified**
   - Treat all claims as provisional
   
2. **Decompose hypothesis into elements**
   - Subject: Who is the actor?
   - Verb: What action occurred?
   - Object: Who/what was affected?
   - Each element requires independent verification

3. **Chronological mapping**
   - Build timeline from open sources first
   - Identify gaps requiring investigation
   
4. **Regular hypothesis testing**
   - After each new evidence item, ask: does this confirm, contradict, or modify the hypothesis?
   - Be willing to abandon hypotheses that don't survive testing

5. **Documentation discipline**
   - Record exactly when and where each evidence item was obtained
   - Maintain chain of custody for digital evidence

### 5.4 Second Antithesis (Externality Framing)

After completing Claims → Contradictions → Resolution, apply ANOTHER antithesis to the resolution itself:

1. **Who bears costs?**
   - The entity generating output may not bear its costs
   - Costs often externalized to maintainers, communities, future generations

2. **Who captures benefits?**
   - Surplus may flow elsewhere than claimed
   - Follow actual benefit distribution, not stated intent

3. **What's multiplied?**
   - Systems often multiply existing asymmetries rather than create new value
   - "X works" may mean "X works for some at expense of others"

**Application example:**
> "AI coding tools increase developer productivity" (synthesis from claims vs contradictions)
> Second antithesis: WHO bears the cost? 12:1 contributor-to-maintainer time ratio discovered.
> Productivity gains externalized as OSS maintainer burden. METR study shows 19% productivity DECREASE for experienced devs.
> → "AI multiplies what you already know" is more accurate than "AI helps everyone code"

*Discovered through vibe coding claims investigation, January 2026.*

### 5.5 Language/Power Analysis

Examine what ideological work terminology does. When corporate or institutional frameworks import nation-state language:

**Recognition patterns:**
- **"Most Favored Nations" (MFN) provisions**: Trade treaty language imported into platform contracts. Borrows legitimacy of international law for private extraction.
- **"Protocol" or "standard"**: Technical neutrality masking unilateral design choices.
- **"Consent"**: Legal term imported where power asymmetry makes meaningful consent impossible.
- **"Universal" anything**: Check whose perspective defines universality.

**Analytical questions:**
- What legitimacy does this terminology borrow?
- What's obscured by using this term instead of alternatives?
- Who benefits from framing it this way?
- What would change if we described the same thing in plain material terms?

**Insight:** Language doing this work often signals normalization of post-state power structures—corporate entities assuming governmental functions without governmental accountability.

*Discovered through Google UCP investigation (MFN provisions), January 2026.*

### 5.6 Bulletproofing (amaBhungane Method)

From South African investigative journalism, for legally defensible analysis:

1. **Data supports story, not vice versa**
   - Never stretch evidence to fit narrative
   - If data doesn't support claim, modify claim

2. **Prosecutorial cross-examination**
   - Subject every major claim to adversarial review
   - What would opposing counsel argue?
   - Where are the weaknesses?

3. **Multiple verification paths**
   - Each critical claim needs 2+ independent verification routes
   - Different source types (documentary + testimonial + physical)

4. **Anticipate legal challenge**
   - Would this survive defamation lawsuit?
   - Are sources protected?
   - Is evidence acquisition defensible?

### 5.7 Comprehensive National Power Assessment (中国方法)

Adapted from Chinese state methodology for corporate entity assessment:

Evaluate entity across **all** dimensions simultaneously rather than sequentially:

| Dimension | Assessment Questions |
|-----------|---------------------|
| **Economic resources** | Revenue scale, profit margins, market position, growth trajectory |
| **Human capital** | Workforce quality, management capability, institutional knowledge |
| **Natural resources** | Resource access, supply chain control, extraction rights |
| **Capital resources** | Financing access, debt structure, investor base |
| **Knowledge/Technology** | IP portfolio, R&D capability, technological moat |
| **Governance resources** | Regulatory relationships, political connections, lobbying capacity |
| **Strategic positioning** | Geopolitical alignment, critical infrastructure role, chokepoint control |
| **Soft power** | Brand value, narrative control, cultural influence |

The insight: **entities weak in one dimension may compensate through strength in others.** Holistic assessment reveals true power configuration.

## 6. Domain Investigation Checklists

### A. Weapons/Military Nexus

**Direct connections:**
- Military contracts (amounts, purposes, recipients, classifications)
- Dual-use technology with weapons applications
- Weapons-complex integration (enrichment, components, systems)
- Export licenses for controlled items
- End-user certificates and their verification

**Historical trajectory:**
- Historical weapons involvement vs. current status
- Trajectory direction (toward or away from weapons work)
- Non-proliferation credentials (treaties, disarmament actions)

**Personnel/revolving door:**
- Board/leadership from DOD/DOE/intelligence agencies
- Advisory boards with security state representation
- Hiring patterns from weapons complex
- Post-employment flows TO weapons complex

**Structural enablement:**
- Does civilian work provide cover for classified work?
- Does entity's existence strengthen weapons infrastructure?
- Would entity's absence degrade weapons capability?

### B. Power Structure & Governance

**Ownership analysis:**
- Ultimate beneficial ownership (trace through layers)
- State ownership percentage and which state
- Institutional investor composition
- Concentrated private control (family, founder)

**Revolving door mapping:**
- Government → Company flows (regulatory capture indicator)
- Company → Government flows (policy influence indicator)
- Intelligence/military alumni in leadership

**Political influence:**
- Lobbying expenditure and targets
- Political donations (corporate and executive personal)
- Regulatory comment submissions
- Trade association memberships
- Think tank funding

**Regulatory relationship:**
- Enforcement action history
- Consent decree / settlement patterns
- Regulatory exemptions obtained
- Self-regulatory body capture

### C. Labor Practices

**Safety record:**
- Fatalities, injuries, near-misses (rates vs. industry)
- OSHA/equivalent violations and fines
- Workers' compensation claims patterns
- Occupational disease incidence

**Labor rights:**
- Union recognition or suppression
- Collective bargaining status
- Strike history and management response
- NLRB/equivalent complaints

**Documented violations:**
- ILO findings
- Court cases and settlements
- DOL investigations
- Wage theft patterns

**For state-owned entities:**
- Owning state's broader labor rights record
- Forced labor in supply chain exposure
- Independent union legality in jurisdiction

**Standpoint check:**
- What do workers say vs. what management says?
- Are worker voices accessible or suppressed?
- What do former employees report?

### D. Environmental Record

**Contamination:**
- Current contamination (documented, ongoing)
- Legacy contamination (historical sites)
- Remediation status and adequacy
- Community health impacts

**Regulatory compliance:**
- Violation history and fines
- Permit exceedances
- Self-reported vs. discovered violations
- Consent decree compliance

**Structural issues:**
- Outstanding environmental liabilities
- Decommissioning/closure cost adequacy
- Climate exposure and transition risk
- Water rights and usage

**Environmental justice:**
- Proximity to marginalized communities
- Disparate impact patterns
- Community consent processes (or lack thereof)
- Indigenous land rights issues

### E. Supply Chain

**Upstream tracing:**
- Raw material origins (country, region, specific sites)
- Extraction conditions (labor, environmental, conflict)
- Processing chain and associated entities
- Chokepoint dependencies

**Downstream tracing:**
- Customer identification (who uses product/service)
- End-use verification (what is it used for)
- Problematic end-user exposure (weapons, surveillance, repression)
- Secondary sales and diversion risk

**Supply chain governance:**
- Audit practices and their independence
- Certification schemes and their credibility
- Disclosed vs. actual supplier lists
- Subcontractor visibility

### F. Geopolitical Positioning

**Alignment assessment:**
- Which power structures does entity serve/strengthen?
- Multi-vector (hedging) vs. aligned positioning
- Strategic significance to various powers
- Critical infrastructure designation(s)

**Structural role:**
- Does entity's existence challenge or reinforce imperial structures?
- Chokepoint or node in strategic networks?
- Substitutability (could alternatives emerge?)
- Dependency creation (does entity create dependencies in others?)

**Apply symmetrically across poles** (see Section 3.3)

### G. Epistemic/Cultural (Often Overlooked)

**Knowledge extraction:**
- Data collection and ownership
- IP acquisition patterns
- Research relationships and knowledge flows
- Traditional knowledge appropriation

**Narrative control:**
- Media ownership or influence
- Platform control and content moderation
- Think tank and academic funding
- PR and reputation management scale

**Cultural dynamics:**
- Cultural appropriation patterns
- Community consultation (genuine vs. performative)
- Local benefit distribution
- Displacement effects

**Apply coloniality analysis:**
- Does entity perpetuate colonial patterns in economy, authority, knowledge?
- Whose knowledge has been made invisible?
- What alternatives have been suppressed?

## 7. State Actor Analysis

### 7.1 Separating Entity from State

**For state-owned or state-connected entities, examine separately:**

1. **Entity-specific behavior**
   - What has this company specifically done?
   - Documented actions, not assumed guilt-by-association
   
2. **State actor behavior**
   - What has the owning/connected state done that's ethically relevant?
   - Document with same rigor as corporate behavior
   
3. **Degree of integration**
   - Does entity operate with genuine independence?
   - Or is it an instrument of state policy?
   - Evidence: leadership appointment, strategic direction, operational control
   
4. **Comparative context**
   - How does state compare to realistic alternatives?
   - Not against ideal, but against actual other options
   - Avoid false equivalence AND Western-centric exceptionalism

### 7.2 State Behavior Categories

Document state behavior across categories with same evidence standards as corporate behavior:

**Domestic practices:**
- Repression of protests, labor, dissent (documented incidents)
- Judicial independence and rule of law
- Media freedom and information control
- Minority/indigenous treatment

**External practices:**
- Military interventions (where, scale, documented harms)
- Economic coercion patterns
- Proxy/client relationships
- International law compliance

**Economic structure:**
- Wealth distribution patterns
- Labor rights framework
- Environmental governance
- Corruption indices (with methodology awareness)

### 7.3 Avoid Common Errors

**False equivalence:** "All states are equally problematic" ignores documented differences in harm scales.

**Western exceptionalism:** "Western states are fundamentally different" ignores documented Western harms.

**Authoritarian essentialism:** "Authoritarian states are inherently worse" substitutes label for analysis of specific behaviors.

**Correct approach:** Document specific behaviors with evidence, compare against specific alternatives relevant to the investment decision, apply consistent standards.

## 8. Recomposition & Output

### 8.1 Recomposition Protocol

After applying all relevant lenses:

1. **Surface emergent patterns**
   - What appears only when viewing domains together?
   - What interactions between domains change assessment?
   
2. **Identify cross-lens contradictions**
   - Company strong on environment but weak on labor—why?
   - Company claims civilian mission but governance shows weapons-complex revolving door—what does this combination reveal?
   
3. **Map system interactions**
   - How do findings in one domain enable or constrain behavior in another?
   - What systemic position does entity occupy?
   
4. **Multi-framework triangulation**
   - Do different analytical traditions reach similar conclusions?
   - Where do they diverge, and what does divergence reveal?
   
5. **Apply user's stated weighting**
   - Produce ranked assessment per their framework
   - Make weighting explicit, not hidden
   
6. **Articulate trade-offs explicitly**
   - What does user gain/lose with each choice?
   - No option may be clean—clarify the compromises

### 8.2 Graduated Assessment Output

**Avoid binary pass/fail. Instead produce:**

1. **Lens-by-lens ranking** of compared entities with evidence quality noted
2. **Explicit trade-offs** between dimensions
3. **"Least problematic" framing** when no clean options exist
4. **Framework-conditional conclusions**: "If X weighs heaviest, then Y; if Z weighs heaviest, then W"
5. **Uncertainty acknowledgment**: What remains unknown? What would resolve it?

## 9. Sector-Specific Patterns

### Weapons/Defense Companies

- Trace ALL government contracts (not just disclosed "defense revenue")
- Investigate dual-use applications of "civilian" technology
- Map revolving door with DOD/DOE/intelligence agencies
- Check for sole-source contracting bypassing competition
- Examine classified work indicators (facility clearances, SCIF presence)
- Apply nonproliferation lens even for nominally civilian nuclear

### Extractive Industries

- Trace full supply chain including processing and beneficiation
- Identify all joint venture partners and their state connections
- Map end-users including potential weapons program supply
- Document environmental justice issues in extraction communities
- Apply dependency theory: who benefits from extraction, who bears costs?
- Check indigenous land rights and consent processes

### State-Owned Entities

- Separate company behavior from state behavior (Section 7)
- Examine state's international positioning (aligned vs. multi-vector)
- Check for genuine operational independence or instrument-of-state status
- Compare state behavior to realistic alternatives, not ideal
- Apply same standards to Western state-connected entities (pension funds, sovereign wealth, development finance)

### Technology/Infrastructure

- Trace data flows and surveillance enablement
- Map platform dependencies and chokepoint control
- Identify content moderation / information control functions
- Examine dual-use potential for military/intelligence applications
- Check data localization and cross-border transfer patterns
- Apply epistemic lens: whose knowledge is extracted, controlled, suppressed?

### Financial Services

- Map regulatory arbitrage structures
- Identify tax haven usage and beneficial ownership opacity
- Check sanctions compliance and enforcement action history
- Examine predatory product exposure (payday lending, aggressive collections)
- Apply dependency theory to credit relationships
- Check for financial warfare vulnerability/participation

## 10. Quality Assurance

### 10.1 Pre-Publication Checklist

Before finalizing any assessment:

- [ ] All major claims have evidence quality ratings
- [ ] Contradicting evidence has been actively sought (not just supporting evidence)
- [ ] Deductive verification applied (what should exist if claims true?)
- [ ] Multiple analytical frameworks considered
- [ ] Analyst positioning acknowledged
- [ ] User's framework explicitly applied (not analyst's preferences)
- [ ] Trade-offs articulated, not hidden
- [ ] Uncertainty explicitly stated
- [ ] Sources would survive adversarial scrutiny

### 10.2 Common Failure Modes

**Confirmation bias:** Only seeking evidence supporting initial hypothesis
→ Remedy: ACH structure, active contradiction search

**Source capture:** Over-relying on cooperative sources with agendas
→ Remedy: Adversarial source inclusion, motivation flagging

**Framework blindness:** Missing what your analytical tradition doesn't ask
→ Remedy: Multi-framework cross-check, standpoint method

**False balance:** Treating all evidence as equally weighted
→ Remedy: Explicit evidence quality ratings, diagnostic value assessment

**Premature synthesis:** Reaching conclusions before all lenses applied
→ Remedy: Strict phase separation, recomposition only after full decomposition

**Western default:** Treating Western methods/sources as objective baseline
→ Remedy: Multi-polar analysis, symmetric standards application

## 11. When to Use Full Multi-Lens Analysis

**Use full methodology when:**
- Comparing 3+ entities across 4+ domains
- User's framework requires specialized domain knowledge
- Findings in one domain may contradict findings in another
- Evidence quality varies significantly across domains
- Synthesis requires explicit trade-off articulation
- Stakes are high (significant investment, public-facing analysis)

**Use simplified direct investigation when:**
- Single entity, single domain query
- User needs quick factual answer, not comparative assessment
- Domain expertise is general (e.g., "is this company profitable?")
- Preliminary screening before full analysis

**Scaling approach:**
- Quick screen: 1-2 lenses, key questions only, 30-minute scope
- Standard analysis: 3-4 lenses, Claims→Contradictions→Resolution, documented sources

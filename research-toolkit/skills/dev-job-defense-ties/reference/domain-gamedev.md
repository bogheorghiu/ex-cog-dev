# Domain overlay: gamedev

> **An overlay, not a replacement.** It *extends* `domain-dev.md` with the tells specific to game-engine / real-time-3D work — where disguised-defense dev roles cluster densest, because the same skills (Unreal/Unity, real-time-3D, simulation) build both consumer games and military trainers. Install it by copying into `<config>/domain/gamedev.md`; the screen reads the default pack **plus** any installed overlays. Everything in `domain-dev.md` still applies — this only adds signal.

## Added lexicon (Step 1)

**Dead giveaways (game-engine mil-sim terms of art):** "synthetic training environment (STE)," "CIGI" (Common Image Generator Interface), "DIS/HLA" (distributed-sim interop protocols), "OneSAF," "constructive simulation," "image generator (IG)," "after-action review (AAR)" in a training-sim context.

**Engine / stack tells in a defense frame:** Unreal Engine / Unity / Unigine / Godot or "real-time 3D" / "game engine" paired with *mission rehearsal*, *trainer*, *sensor stimulation*, *geospecific terrain*, or *whole-earth / 3D terrain* — the civilian engine is the tell only when the *use* is military.

**The studio→sim pipeline:** a games studio (or ex-games talent) "pivoting to simulation," "applying game tech to training," or "serious games" for a government/defense customer. "Serious games" is the euphemism that most often launders a gamedev role into a trainer.

## Added buyers (Step 3) — sim integrators & game-tech-for-defense

These are where gamedev skills actually land in defense, often one layer below a prime:

- **Bohemia Interactive Simulations (BISim) / VBS** (the VBS franchise — a games studio's engine turned military trainer; the archetype).
- **MÄK / VT-MÄK**, **Improbable** (defense/M2 division), **Hadean**, **Cole Engineering Services (CESI / SE Core)**, **Diligent / ECS**, **CAE** (training & simulation), **MASA**, **Calytrix**.
- Plus the `domain-dev.md` primes when they staff their *training & simulation* lines (e.g. Lockheed, Leidos, Booz Allen training divisions) — gamedev talent usually enters there.

## End-use note

Gamedev defense roles concentrate at the **TRAINING/SIM** rung of the end-use ladder (trainers, visualization, IG) — the benign-looking end. Under the conduct test that still counts as military end-use; the threshold decides whether the benign rung is acceptable for the buyer in question.
